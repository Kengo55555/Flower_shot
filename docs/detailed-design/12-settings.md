# 12. 設定画面

依存関係: 03（認証フロー）

---

## 概要

位置情報のデフォルト設定、ストレージ容量の確認、ログアウト機能を提供する。保護者向けの情報も表示する。

---

## 実装タスク

### 12.1 設定画面（src/pages/SettingsPage.tsx）

- [ ] 画面レイアウト（セクション区切りのリスト形式）
  - セクション1: アカウント情報
  - セクション2: 位置情報設定
  - セクション3: ストレージ情報
  - セクション4: アプリ情報
  - セクション5: ログアウト

### 12.2 アカウント情報セクション

- [ ] 表示内容
  - Googleアカウントのアイコン（`photoURL`）
  - 表示名（`displayName`）
  - メールアドレス（`email`）
- [ ] 編集機能はなし（Google アカウントの情報をそのまま表示）

### 12.3 位置情報設定セクション

- [ ] 「ばしょを きろくする（きほん）」トグル
  - 現在の設定値: Firestore `users/{uid}.settings.locationDefaultOn`
  - トグル変更時に Firestore を更新
  - ラベル: 「さつえいするとき ばしょを きろくする」（RubyText）
  - 説明文: 「ONにすると おはなを みつけた ばしょを ちずに のこせるよ」

### 12.4 ストレージ情報セクション

- [ ] 表示内容（要件定義書 §7.2 準拠）
  ```
  しゃしん: {N}まい ({usageMB}MB / やく1GB)
  きろく: {N}けん
  きょうの はんてい: {todayCount}かい / 30かい
  ```
- [ ] `getStorageEstimate()` で IndexedDB の使用量を取得
- [ ] `useUsageLimit()` で当日の判定回数を取得
- [ ] `useRecords()` で記録件数を取得
- [ ] 容量が800MB超の場合: 警告メッセージ表示（オレンジ背景）
  - 「しゃしんを ほぞんできるばしょが すくなくなってきました」

### 12.5 古い画像の整理機能

- [ ] 「ふるい しゃしんを せいりする」ボタン
- [ ] タップ → 削除対象の選択画面
  - 日付順に画像一覧を表示（古いものから）
  - チェックボックスで複数選択
  - 「えらんだ しゃしんを けす」ボタン
- [ ] 削除処理
  - IndexedDB から画像を削除
  - Firestore のレコード自体は残す（テキスト情報は保持）
  - 削除後にストレージ情報を再取得して表示更新
- [ ] 確認ダイアログ: 「えらんだ しゃしんを けしてもいい？（きろくは のこるよ）」

### 12.6 アプリ情報セクション

- [ ] 表示内容
  - アプリ名: Flower Shot
  - バージョン: 1.0.0
  - 「つかいかた」リンク → TutorialPage
  - PWAインストール状態: 「ホームがめん ついか: ずみ / まだ」
    - 未インストール時: 「ついかする」ボタン → TutorialPage

### 12.7 ログアウトセクション

- [ ] 「ログアウト」ボタン（画面最下部）
  - ボタンスタイル: 控えめな赤系テキストボタン
  - タップ → 確認ダイアログ「ログアウトしますか？」
  - 確認 → `signOutUser()` → `/login` にリダイレクト

### 12.8 安全注意喚起

- [ ] アプリ起動時（HomePage表示時）に初回のみ表示
  - 「おそとで つかうときは あんぜんな ところで つかおうね」
  - 小さなモーダル or トースト表示
  - 「わかった！」ボタンで閉じる
  - 表示済みフラグを `localStorage` に保存

---

## 完了条件

- アカウント情報が正しく表示される
- 位置情報のデフォルト設定を変更でき、Firestoreに反映される
- ストレージ使用量・記録件数・判定回数が正しく表示される
- 容量警告が800MB超で表示される
- 古い画像を選択して削除できる
- ログアウトが正しく動作する
- PWAインストール状態が表示される
