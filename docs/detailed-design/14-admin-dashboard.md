# 14. 管理者ダッシュボード（Phase 2）

依存関係: 03（認証フロー）、07（保存機能）、08（利用回数制限）

---

## 概要

管理者（nomurakengo@gmail.com）専用の管理画面。ユーザー管理、API使用量の監視、ブロック機能を提供する。

**v2.0変更点**: 設定画面の「管理者」セクションからアクセス。ダッシュボードにユーザー一覧を統合表示。「もどる」ボタンで設定画面に戻る。

---

## 実装タスク

### 14.1 管理者ダッシュボード（src/pages/admin/AdminDashboard.tsx）

- [x] 画面レイアウト
  - ヘッダー: 「かんりしゃ」+ 「もどる」ボタン（→ `/settings`）
  - サマリーカード（3列グリッド）:
    - ユーザー数
    - 撮影記録数（totalRecords）
    - 本日のAPI使用量（{globalCount}/500）
  - 未確認ユーザー警告バナー
  - ユーザー一覧（確認・ブロック・ブロック解除ボタン付き）

### 14.2 ダッシュボードのデータ取得

- [x] 全ユーザー取得（`getAllUsers()`）
- [x] 各ユーザーのレコード数を集計
- [x] 各ユーザーのブロック状態を取得（`checkBlocked()`）
- [x] 本日のAPI使用量（`getGlobalUsage()`）

### 14.3 ユーザー一覧（ダッシュボード内統合）

- [x] ユーザーカード形式で表示
  - アイコン・表示名・メールアドレス
  - 登録日・撮影数
  - 未確認ユーザーはオレンジのborder-left
  - ブロック中は半透明
- [x] アクションボタン: 確認・ブロック・ブロック解除

### 14.4 ユーザー確認機能

- [x] 「かくにん」ボタンで `reviewedByAdmin` を `true` に更新

### 14.5 ブロック機能

- [x] 「ブロック」ボタンで `blocked_users` コレクションにドキュメント追加

### 14.6 ブロック解除機能

- [x] 「ブロックかいじょ」ボタンで `blocked_users` からドキュメント削除

### 14.7 撮影記録一覧画面（src/pages/admin/AdminRecords.tsx）

- [x] 全撮影記録のテキスト情報を閲覧可能

### 14.8 ユーザー一覧画面（src/pages/admin/AdminUsers.tsx）

- [x] 全ユーザーの詳細一覧

### 14.9 管理者画面のルーティング

- [x] `/admin` → AdminDashboard
- [x] `/admin/users` → AdminUsers
- [x] `/admin/records` → AdminRecords
- [x] すべて `AuthGuard` + `AdminGuard` で保護
- [x] 「もどる」ボタン → `/settings`
