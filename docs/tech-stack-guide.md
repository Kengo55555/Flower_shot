# Flower Shot 技術スタックガイド

**この資料の目的**: Flower Shot で使われている技術を、エンジニアでない方にもわかるように説明します。

最終更新日: 2026-05-01

---

## はじめに

Flower Shot は「道ばたの花をスマホで撮ると、AIが名前を教えてくれるアプリ」です。子ども（小学1年生くらい）でも使えるシンプルなアプリですが、裏側ではたくさんの技術が協力して動いています。

この資料では、それぞれの技術が **何をしているのか** を、できるだけやさしい言葉で説明します。

---

## 全体のしくみ（データの流れ）

Flower Shot を使うとき、裏側では以下のようなことが起きています。

```
お子さんが花を撮影
    |
    v
[スマホの中のアプリ] ---- インターネット ----> [花のAI（Pl@ntNet）]
 (React で作った画面)                           「これはタンポポです（85%）」
    |                                                |
    |  <------ 判定結果が返ってくる -----------------+
    |
    v
[花の名前を保存]               [写真を保存]
 Firestore（クラウドの棚）       IndexedDB（スマホの中の棚）
    |
    v
[もっとくわしく]  -----------> [Wikipedia]
                                「タンポポは キク科の植物で...」
    |
    v
[地図に記録]  ----------------> [OpenStreetMap の地図]
                                 撮影した場所にピンを表示
```

ポイントは **「写真はスマホの中だけに保存」「花の名前などの文字情報だけがインターネット上に保存」** という点です。これにより、クラウドの保存容量を節約しつつ、プライバシーも守れます。

## 各技術の説明

### 1. React + TypeScript（画面を作る技術）

**ひとことで言うと**: アプリの「見た目」と「動き」を作るための道具です。

- **React**（リアクト）は、スマホやパソコンの画面に表示されるボタン・文字・写真などを組み立てるための技術です。Facebook（現Meta）が作りました。
  - たとえば「撮影ボタンを押したら、カメラが起動する」「判定結果を画面に表示する」といった動きを、Reactが担当しています。
  - 画面の部品（ボタン、カードなど）を「コンポーネント」（部品のこと）として作り、組み合わせて画面全体を作ります。Flower Shot では、ホーム画面、カメラ画面、図鑑画面などがそれぞれコンポーネントの組み合わせでできています。

- **TypeScript**（タイプスクリプト）は、プログラムの書き間違いを事前に見つけてくれる仕組みです。
  - 通常の JavaScript（ジャバスクリプト＝ウェブの標準プログラミング言語）に「型チェック」（データの種類を事前に決めておくこと）を追加したものです。
  - たとえば「花の名前は文字のはずなのに、数字が入ってしまった」というミスを、プログラムを動かす前に検出できます。

**このアプリでの役割**: Flower Shot の画面すべて（ホーム、カメラ、結果表示、図鑑、地図、設定など）を作っています。

---

### 2. Vite（開発ツール）

**ひとことで言うと**: プログラムを書いてからアプリとして動かすまでの「翻訳・梱包」をしてくれる道具です。

- 開発者がプログラムを書くとき、ブラウザ（Safari や Chrome）がそのまま理解できる形ではありません。TypeScript や React の特殊な書き方を、ブラウザが理解できる形に変換（ビルドと呼びます）する必要があります。
- **Vite**（ヴィート、フランス語で「速い」の意味）は、この変換作業をとても高速に行ってくれるツールです。
  - 開発中にプログラムを書き換えると、ほぼ瞬時に画面に反映されます（ホットリロードと呼ばれる機能）。
  - アプリを公開するときには、すべてのファイルを1つにまとめて最適化してくれます。

**このアプリでの役割**: 開発者がプログラムを効率的に書き、最終的にアプリとして公開できる形にまとめるために使っています。

---

### 3. Tailwind CSS（デザイン技術）

**ひとことで言うと**: アプリの「見た目のデザイン」（色・大きさ・余白など）を効率よく指定するための道具です。

- ウェブアプリの見た目を決めるには **CSS**（シーエスエス＝ウェブページの見た目を指定する言語）を使います。
- **Tailwind CSS**（テイルウィンド シーエスエス）は、CSSを書きやすくする道具です。「この文字は大きくして、この背景はピンクにして」といった指定を、短い名前（クラス名）をつけるだけで実現できます。
  - たとえば `text-2xl`（文字を大きく）、`bg-pink-500`（背景をピンクに）、`rounded-lg`（角を丸く）のように書きます。
- Flower Shot では、子どもが使いやすい大きなボタンやカラフルな配色を、この技術で実現しています。

**このアプリでの役割**: ボタンの色、文字の大きさ、カードのレイアウトなど、アプリの見た目すべてを担当しています。

---

### 4. Firebase Authentication（ログイン）

**ひとことで言うと**: 「Googleアカウントでログインする」機能を提供してくれるサービスです。

- **Firebase**（ファイヤーベース）は、Google が提供するアプリ開発の支援サービスです。
- その中の **Authentication**（オーセンティケーション＝本人確認）という機能を使うと、「Googleでログイン」ボタンを簡単に実装できます。
- ログインすることで、「誰の撮影記録か」を区別でき、他の人の記録が混ざることを防げます。
- Flower Shot では、保護者のGoogleアカウントを子どもと共用する想定です。

**このアプリでの役割**: ログイン機能と、不適切な利用者をブロックする仕組みの土台になっています。

---

### 5. Cloud Firestore（データベース）

**ひとことで言うと**: 花の名前や撮影日時などの「文字情報」をインターネット上に保存する棚です。

- **Cloud Firestore**（クラウド ファイヤーストア）は、Firebase が提供するデータベース（データを整理して保存する場所）です。
- 保存しているもの:
  - 花の名前（例: 「タンポポ」）
  - 学名（例: 「Taraxacum」）
  - 撮影日時
  - 撮影場所の座標（緯度・経度）
  - 判定の信頼度（「85%の確率でタンポポ」など）
  - ユーザー情報（誰がログインしたか）
  - 1日に何回花を判定したかのカウント
- 写真そのものは保存していません（容量が大きいため）。写真はスマホの中だけに保存されます。

**このアプリでの役割**: アプリの「記憶」を担当しています。スマホを変えても、ログインすれば花の名前や記録は引き継がれます（ただし写真は引き継がれません）。

---

### 6. IndexedDB（端末内画像保存）

**ひとことで言うと**: 撮影した写真をスマホの中だけに保存する「アルバム棚」です。

- **IndexedDB**（インデックスト ディービー）は、ウェブブラウザの中にある「大きなデータ保存庫」です。スマホの中だけにデータが残り、インターネット上にはアップロードされません。
- Flower Shot では、撮影した花の写真をすべてこの仕組みで保存しています。
- iPhoneの場合、約1GB（500枚程度）まで保存できます。
- **注意点**: アプリを削除すると写真も消えます。また、別のスマホからは写真を見ることができません。

**このアプリでの役割**: 写真をクラウドに送らないことで、通信量の節約とプライバシーの保護を両立しています。

---

### 7. Pl@ntNet API（花の判定AI）

**ひとことで言うと**: 花の写真を送ると「これは何の花か」を教えてくれるAIサービスです。

- **Pl@ntNet**（プラントネット）は、フランスの研究機関が開発した植物識別のAIです。世界中の研究者と市民が集めた植物写真のデータをもとに、写真から植物の種類を判定します。
- **API**（エーピーアイ＝ソフトウェア同士がやり取りする窓口）を通じて、花の写真を送ると「この花はタンポポで、信頼度は85%です」といった結果が返ってきます。
- 非商用（お金儲け目的でない利用）なら無料で使えます。ただし、1日に500回までという制限があるため、Flower Shot ではユーザー1人あたり1日100回までに制限しています。

**このアプリでの役割**: アプリの「目」にあたる部分で、花の名前を特定する一番大事な機能です。

---

### 8. Wikipedia API（花の詳細情報）

**ひとことで言うと**: 判定された花の詳しい説明を、Wikipedia から取ってきて表示する仕組みです。

- Wikipedia（ウィキペディア＝誰でも編集できるオンライン百科事典）の情報を、アプリの中に取り込んで表示しています。
- 花の名前で Wikipedia を検索し、要約文と画像を取得します。
- 完全に無料で、APIキー（利用するための鍵）も不要です。

**このアプリでの役割**: 「もっとくわしく」ボタンを押したときに、花の説明や特徴を表示します。自前で花のデータベースを持つ必要がなく、運用の手間がかかりません。

---

### 9. Leaflet + OpenStreetMap（地図）

**ひとことで言うと**: アプリ内で地図を表示し、花を見つけた場所にピンを立てる仕組みです。

- **Leaflet**（リーフレット）は、ウェブページに地図を埋め込むための無料の道具です。
- **OpenStreetMap**（オープンストリートマップ、略称 OSM）は、世界中のボランティアが作成している無料の地図データです。Google マップのような有料サービスと違い、APIキーや利用料金が不要です。
- Flower Shot では、Leaflet と OpenStreetMap を組み合わせて以下を実現しています:
  - 「ちず」タブで、今まで花を見つけた場所をピンで表示
  - 花を判定した後、「ちずで えらぶ」ボタンで撮影場所を地図上で指定

**このアプリでの役割**: 撮影記録を地図で振り返ったり、撮影場所を記録したりする機能を担当しています。

---

### 10. Cloudflare Pages（アプリの公開）

**ひとことで言うと**: 作ったアプリをインターネット上に公開して、誰でもアクセスできるようにするサービスです。

- **Cloudflare Pages**（クラウドフレア ページズ）は、ウェブサイトやウェブアプリを公開するためのサービスです。
- 世界中にサーバー（データを配信するコンピュータ）があり、アクセスした人の近くのサーバーからデータが届くため、表示が速いのが特徴です。
- 通信量（データの転送量）が無制限で無料です。小規模なアプリには非常に適しています。

**このアプリでの役割**: Flower Shot のアプリ本体（画面やプログラム）をインターネット上に置いて、スマホからアクセスできるようにしています。

---

### 11. Cloudflare Pages Functions（APIの中継）

**ひとことで言うと**: 花の判定AIとアプリの間に立って、大事な「鍵」を守る中継役です。

- Pl@ntNet API を使うには、**APIキー**（利用するための秘密の鍵）が必要です。この鍵がもし外部に漏れると、他人に勝手に使われてしまう危険があります。
- **Cloudflare Pages Functions**（クラウドフレア ページズ ファンクションズ）は、サーバー上で小さなプログラムを動かす仕組みです。
- Flower Shot では、この仕組みを「中継役」として使っています:
  1. スマホのアプリが花の写真を Cloudflare に送る
  2. Cloudflare がAPIキーを付けて Pl@ntNet に転送する
  3. Pl@ntNet からの結果を Cloudflare がアプリに返す
- こうすることで、APIキーがスマホ側に露出せず、安全に利用できます。

**このアプリでの役割**: セキュリティ上の「守衛」のような存在です。1日10万リクエストまで無料で使えます。

---

### 12. jsPDF（PDFアルバム生成）

**ひとことで言うと**: 撮影した花の写真と記録を、PDF（印刷できるファイル形式）のアルバムにまとめる道具です。

- **jsPDF**（ジェーエス ピーディーエフ）は、ブラウザの中でPDFファイルを作れる無料の道具です。
- サーバーに頼らず、スマホの中だけでPDFを生成するため、追加のコストがかかりません。
- Flower Shot では、図鑑画面から「アルバムを つくる（PDF）」ボタンを押すと、以下のようなPDFが作られます:
  - 表紙（アプリ名、年、花の数）
  - 本文（1ページに2枚ずつ、写真・花名・学名・日付・信頼度を記載）
- 作ったPDFは印刷したり、家族に共有したりできます。端末の故障に備えた写真のバックアップとしても役立ちます。

**このアプリでの役割**: 撮影した花の「思い出アルバム」を作る機能を実現しています。

---

### 13. PWA（スマホアプリ化）

**ひとことで言うと**: ウェブサイトを「まるでスマホアプリのように」使えるようにする技術です。

- **PWA**（ピーダブリューエー＝Progressive Web App の略、「進化したウェブアプリ」の意味）は、ウェブサイトをスマホのホーム画面に追加して、アプリのように使える仕組みです。
- App Store からダウンロードする通常のアプリとは違い、Safari でウェブサイトにアクセスして「ホーム画面に追加」するだけでインストールできます。
- PWA として使うメリット:
  - ホーム画面にアイコンが表示され、タップするだけで起動できる
  - ブラウザの上下のバー（アドレスバーなど）が消えて、全画面で使える
  - iOS では、ホーム画面に追加しておくとデータが消えにくくなる（通常のSafari だと7日間アクセスしないとデータが消える場合がある）

**このアプリでの役割**: App Store での公開手続きなしに、スマホアプリと同じ使い心地を実現しています。初回にチュートリアル画面で「ホーム画面に追加」の方法を案内しています。

## 技術同士のつながり（まとめ図）

以下に、各技術がどう連携しているかを示します。

```
[お子さんのスマホ（iPhone / iPad）]
|
|  ホーム画面からアプリを起動（PWA）
|
+-- [アプリの画面]  React + TypeScript で構築
|   |               Tailwind CSS でデザイン
|   |               Vite でビルド（開発時の道具）
|   |
|   |-- 撮影 ----> 写真を IndexedDB に保存（スマホの中）
|   |
|   |-- 判定 ----> Cloudflare Pages Functions（中継役）
|   |                  |
|   |                  +----> Pl@ntNet API（花のAI）
|   |                           「タンポポです（85%）」
|   |
|   |-- 保存 ----> Cloud Firestore（花の名前・日付などの文字情報）
|   |
|   |-- 詳細 ----> Wikipedia API（花の説明を取得）
|   |
|   |-- 地図 ----> Leaflet + OpenStreetMap（撮影場所を表示）
|   |
|   |-- PDF  ----> jsPDF（アルバムを生成）
|   |
|   +-- ログイン -> Firebase Authentication（Googleアカウント）
|
+-- [Cloudflare Pages] アプリのファイルを配信（アプリの公開場所）
```

## よくある質問

### Q. なぜ App Store のアプリにしなかったのですか？
A. App Store で公開するには、年間約15,000円の開発者登録費がかかり、審査にも時間がかかります。Flower Shot は家族5人程度で使う小規模なアプリなので、PWA（ウェブアプリをスマホアプリ風に使う技術）を使うことで、費用ゼロ・即日公開を実現しました。

### Q. 写真がクラウドに保存されないのはなぜですか？
A. クラウドに写真を保存すると、保存容量に応じた費用が発生します。500枚の写真だと約1GBになり、無料枠をすぐに超えてしまいます。花の名前や日時はテキスト（文字）だけなので容量が非常に小さく、無料枠の範囲で長期間運用できます。

### Q. すべて無料で使えるのですか？
A. はい。Flower Shot で使っているすべての技術は、この規模（5人程度）であれば無料の範囲で利用できます。ただし、花の判定AI（Pl@ntNet）には1日500回までの制限があります。

### Q. インターネットに接続していないと使えませんか？
A. 花の判定には Pl@ntNet API との通信が必要なので、インターネット接続が必要です。過去の記録（図鑑や地図）の閲覧には、データがスマホ内に保存されていれば見ることができます。
